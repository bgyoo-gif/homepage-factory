How It Works

Sensitive data never reaches the LLM.

LLM Capsule sits between your application and the LLM API. Every prompt is intercepted, inspected, and cleaned before transmission.

RAW PROMPT

Input — before LLM Capsule

Summarize the claim filed by John Martinez (ID: P-8847211) on March 3, 2025. The patient's address is 42 Elm Street, Chicago, IL 60601. Diagnosis code: ICD-10: J45.20.
↓ LLM Capsule removes blockers / anonymizes / enables AI
ANONYMIZED PROMPT

Sent to LLM — PII replaced with tokens

Summarize the claim filed by [PERSON_01] (ID: [ID_01]) on [DATE_01]. The patient's address is [ADDRESS_01]. Diagnosis code: [MED_CODE_01].
↓ LLM processes anonymized prompt / returns response
RE-MAPPED OUTPUT

Output — tokens re-mapped for downstream use

The claim filed by [PERSON_01] on [DATE_01] relates to a respiratory condition. The case is pending review. All PII tokens are preserved in the anonymization map for authorized downstream access.
