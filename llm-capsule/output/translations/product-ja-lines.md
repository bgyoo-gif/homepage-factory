LLM Capsule enables enterprise AI adoption by keeping raw data inside your environment, preserving document structure and business context during AI processing, and restoring usable outputs through local restoration — so enterprise teams can safely use any LLM on real documents in production workflows.
LLM Capsuleは、元データを環境内に保持したまま、AI処理中の文書構造とビジネスコンテキストを保存し、ローカル復元によりすぐに使用可能な出力を生成します — 企業チームが本番ワークフローで実際の文書にあらゆるLLMを安全に接続できます。

## Section 06: Structure-Preserving

Beyond text — keep document structure
テキストを超えて — 文書構造を

intact
そのまま維持します

Enterprise workflows do not run on plain text alone. They rely on reports, PDFs, spreadsheets, diagrams, presentations, tables, and mixed-format documents.
企業ワークフローは単純なテキストだけで動いているわけではありません。レポート、PDF、スプレッドシート、ダイアグラム、プレゼンテーション、テーブル、混合形式の文書に依存しています。

Flat masking treats every sensitive value identically, collapsing entity relationships and breaking table schemas. Structure-preserving processing maintains entity consistency across entire documents, preserves table column relationships for accurate extraction, and keeps cross-reference links intact. This is document-aware protection — not flat text anonymization.
単純なマスキングはすべての機密値を同一に処理し、エンティティ関係を崩壊させ、テーブルスキーマを破壊します。構造保存処理は文書全体にわたるエンティティの一貫性を維持し、正確な抽出のためのテーブル列関係を保存し、相互参照リンクを保持します。文書を理解する保護 — 単純なテキスト匿名化ではありません。

## Section 13: FAQ

Common
よくある

Questions
ご質問

Does raw data ever leave the environment?
元データが環境の外に出ることはありますか？

No. LLM Capsule encapsulates sensitive entities locally before any data leaves the enterprise environment. The external AI model processes only the protected capsule — raw original data never crosses the trust boundary. The mapping between original and encapsulated values is stored entirely within your environment and never transmitted externally.
いいえ。LLM Capsuleはデータが企業環境を離れる前に機密エンティティをローカルでカプセル化します。外部AIモデルは保護されたカプセルのみを処理します — 元データは信頼境界を越えることはありません。元の値とカプセル化された値のマッピングは貴社の環境内に完全に保存され、外部に送信されることはありません。

Where does restoration happen?
復元はどこで行われますか？

Restoration happens entirely inside the customer-controlled environment after AI processing completes. The locally stored mapping is applied to AI outputs automatically, restoring original names, figures, dates, and references into the results. This produces enterprise-ready outputs that are directly usable in business workflows — no manual reconstruction required.
復元はAI処理完了後、顧客管理環境の内部で完全に行われます。ローカルに保存されたマッピングがAI出力に自動的に適用され、元の名前、数値、日付、参照番号が結果に復元されます。ビジネスワークフローで直接使用可能なエンタープライズ対応の出力が生成されます — 手動の再構成は不要です。

Can we define our own sensitive entities?
独自の機密エンティティを定義できますか？

Yes. LLM Capsule supports enterprise context control so teams can define business-specific confidential markers beyond standard PII categories. This includes internal identifiers, project names, customer-specific terms, contract references, financial figures, vulnerability labels, and any organization-specific sensitivity category.
はい。LLM CapsuleはEnterprise Context制御をサポートしており、標準PIIカテゴリを超えてビジネス固有の機密マーカーを定義できます。内部識別子、プロジェクト名、顧客固有の用語、契約参照番号、財務数値、脆弱性ラベル、組織固有の機密性カテゴリを含みます。

Does it support document-heavy workflows?
文書中心のワークフローをサポートしていますか？

Yes. LLM Capsule supports structure-preserving processing across enterprise document types including PDFs, Word documents, spreadsheets, tables, presentations, reports, and mixed-format business content. Document structure, entity relationships, and cross-references are maintained during encapsulation.
はい。LLM CapsuleはPDF、Word文書、スプレッドシート、テーブル、プレゼンテーション、レポート、混合形式のビジネスコンテンツを含むエンタープライズ文書タイプ全般にわたる構造保存処理をサポートしています。カプセル化の過程で文書構造、エンティティ関係、相互参照が維持されます。

Can it fit existing enterprise systems?
既存のエンタープライズシステムに適合できますか？

Yes. LLM Capsule is built as a deployable component and supports API and SDK-based integration into existing enterprise portals, partner backends, customer-facing applications, and internal workflow platforms. It wraps existing AI integrations with encapsulation and restoration at the data layer.
はい。LLM Capsuleはデプロイ可能なコンポーネントとして構築されており、既存のエンタープライズポータル、パートナーバックエンド、顧客向けアプリケーション、内部ワークフロープラットフォームへのAPIおよびSDKベースの統合をサポートします。データレイヤーでカプセル化と復元により既存のAI統合をラップします。

Is AWS Marketplace deployment available?
AWS Marketplaceデプロイメントは利用可能ですか？

Yes. LLM Capsule is available on AWS Marketplace for streamlined enterprise procurement, billing, and VPC deployment. This enables rapid deployment within existing AWS environments with standard enterprise procurement workflows.
はい。LLM CapsuleはAWS Marketplaceで簡素化されたエンタープライズ調達、請求、VPCデプロイメントのために提供されています。標準的なエンタープライズ調達ワークフローで既存のAWS環境内に迅速にデプロイできます。

Can it run in air-gapped or on-premise environments?
エアギャップまたはオンプレミス環境で実行できますか？

Yes. LLM Capsule supports on-premise, air-gapped, cloud, hybrid, and embedded deployment models. For air-gapped environments, documents are encapsulated on the isolated network, transferred through controlled channels for AI processing, and results are transferred back for local restoration.
はい。LLM Capsuleはオンプレミス、エアギャップ、クラウド、ハイブリッド、エンベデッドデプロイメントモデルをサポートしています。エアギャップ環境では、隔離ネットワーク上で文書をカプセル化し、制御されたチャネルを通じてAI処理のために転送し、結果を戻してローカルで復元します。

What AI models does LLM Capsule support?
LLM Capsuleはどのようなモデルをサポートしていますか？

LLM Capsule is model-agnostic. Because it operates at the data layer — not the model layer — it works with any external AI service including ChatGPT, Claude, Gemini, Perplexity, or any LLM API. Protection remains stable even when model vendors change.
LLM Capsuleはモデル非依存です。モデルレイヤーではなくデータレイヤーで動作するため、ChatGPT、Claude、Gemini、Perplexity、またはあらゆるLLM APIを含むすべての外部AIサービスと互換性があります。モデルベンダーが変わっても保護は安定的に維持されます。
